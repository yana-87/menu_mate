# Menu Mate (献立提案アプリ)

## 1. アプリ概要と目的

料理のレパートリーがあるにも関わらず、「毎日の献立を考える時間」にストレスを感じているユーザーをターゲットとした、決断疲れを解消するための献立提案アプリ。

ボタン一つで「自分の作れる料理」だけからランダムに提案し、「迷う時間」をゼロにすることを目的とします。

## 2. 実装機能　 （Ver 1.0 - MVP）

- [x] 料理の登録機能（名前とジャンルを保存）
- [x] 登録した料理の一覧表示機能
- [x] 料理の削除機能
- [x] ガチャ機能：登録メニューからランダムに一つ提案

## 2.1 実装機能 (Ver 1.1 デプロイ)

- [] 本番環境へのデプロイ
- [] スマホ対応（レスポンシブ）
- [] 料理の編集機能：登録後のメニュー名やジャンルの修正機能
- [] UI/UX 改善：CSS フレームワーク（Bootstrap 等）を導入し、視認性を向上
- [] ジャンル絞り込み機能の強化：料理の提案の際に「麺類」「揚げ物」「和食」「洋食」など絞り込めるジャンルを増加
- [] バリデーション強化：空欄登録や重複登録を防ぐエラーハンドリングの実装

## 4. 将来的な拡張構想 (Ver 2.0)

- 提案アルゴリズムの改善（前日のジャンルを考慮した選出）
- 「揚げ物、麺類、和食、洋食、中華」など他ジャンルの追加
- 「冷蔵庫の余り物」フィルター:「キャベツ」「豚肉」など、今ある食材を 1 つ選ぶと、それを使える過去のレパートリーだけを表示します。
- 「久しぶり」順の表示:「最近作っていない順」に並べ替える機能。マンネリ化を防ぎ、「あ、これ忘れてた！久々に作ろう」という気づきを与えます。
- 直近の被り防止（カレンダー）:「3 日前にカレーだったから今日は除外」など、自動で直近のメニューを候補から外すロジック。
- 1 週間の献立ドラッグ＆ドロップ:時間がある週末に、レパートリー一覧からカレンダーにポイポイと配置して、1 週間分を埋めてしまう機能。
- 3 人分の分量メモ:レシピ本来の分量ではなく、「我が家（3 人分）なら肉は 300g」といった、家庭ごとの適正分量をメモできる欄（次回の買い物で迷わないため）。
- 季節の提案:自分のレパートリーの中から「去年、寒くなってきた頃にこれを作っていましたよ（おでん等）」とリマインドしてくれる機能。
